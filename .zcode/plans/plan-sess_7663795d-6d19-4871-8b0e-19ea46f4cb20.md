# AOCC V3 — Decision Intelligence Layer

## Confirmed Decisions
- **State persistence**: Client-only via `localStorage` (keyed by user). Zero backend changes.
- **Integration**: New tab toggle at top of AOCC — *"Operational View" (existing 10 widgets)* ↔ *"Decision Intelligence" (new widgets)*. Existing widgets untouched.
- **Scope**: Full spec — all 17 decision types + all 9 bulk actions.

## Architecture: Pure Frontend Layer, No Backend Changes

All computation runs **client-side** from existing React Query data (already fetched by `AoccLayout`). Decisions are *derived* from the existing `ActionItem[]`, `EmployeeCorrelation[]`, `DepartmentHealth[]`, and `RawDataBundle`. No new API calls, no new Firebase paths.

## Key Design Decisions

**Decision derivation (no duplication)**: `generateDecisions()` consumes the *already-computed* `actions`, `correlations`, `departments` from the existing pipeline. It groups related actions into single decisions (Part 6: never duplicate issues). E.g. employee with overdue CAPA + open complaint + declining trend → ONE "Manager Review Required" decision.

**17 Decision Types** — each has a deterministic detector (Immediate Intervention: riskLevel=critical+openActions≥3; SLA Breach: `getSLAInfo()` state='critical'; Complaint Escalation: critical complaint unresolved >24h; Deadline Risk: SLA state='warning'; etc.)

**9-Component Score** — weighted composite, all normalized 0-100: priority 0.20, businessImpact 0.20, urgency 0.18, confidence 0.10, risk 0.10, employeeImpact 0.07, deptImpact 0.07, slaImpact 0.08.

**Predictive Alerts (no AI)** — pure trajectory math: linear extrapolation of trends from existing historical data. "Likely SLA breach" = daysRemaining≤2 AND progress%<50%. Probability = heuristic blend 0-100%.

**Lifecycle + localStorage** — 9 states (new→acknowledged→assigned→in_progress→waiting/escalated→resolved/dismissed→archived). Keyed by `erp_decision_states_<userId>`. Computed decisions default to 'new'; previously-seen retain status.

**Bulk Actions** — each maps to effect: assign (employee picker), review (→acknowledged), close (→resolved), notify (create follow-up via existing hook), escalate (→escalated), open CAPA (existing hook), create follow-up (existing hook), export (client-side CSV — no new API), archive.

**Command Bar** — uses `cmdk` (already installed). Ctrl+K opens palette with preset filters.

**Virtualization** — custom windowing for inbox (render only visible + buffer), avoids new dependency.
